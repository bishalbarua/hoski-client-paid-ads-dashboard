Section 1 of 4: Management Layer

CMO, Marketing Director, Project Manager — purpose, inputs, outputs, chain of command

Chief Marketing Officer
/cmo
NEW

Purpose

Agency-level strategic view only. Never touches individual campaign execution. Answers: how is the agency performing, where should we focus, which clients need attention, what's the growth story.

Decisions it owns

  • Cross-client budget and resource prioritization
  • Which clients to scale, stabilize, or flag
  • Agency-level reporting (investor/partner-facing)
  • Quarterly priorities across the book of business
  • New channel expansion decisions (TikTok, LinkedIn)

Inputs

  • MCC rollup data (all client snapshots)
  • Monthly reports from all clients
  • Weekly check summaries (flagged items)
  • Agency targets / growth goals

Outputs

  • Cross-client health scorecard
  • Quarterly priority list (P1/P2/P3 by client)
  • Agency performance narrative
  • Escalation decisions to Marketing Director

Does NOT do

  • Touch individual campaigns or ad copy
  • Replace the Marketing Director for client work
  • Write client-facing communication

REFACTOR

What changes from current version

  • Currently invokes raw specialists directly (e.g. google-keyword-intelligence). After refactor: delegates to Strategist and Manager layer instead.
  • Currently has no explicit "reports to CMO" awareness. After refactor: escalates cross-client patterns upward.
  • Scope filter tightened: only single-client work. Cross-client decisions escalate to CMO.

What stays the same

  • Dependency graph enforcement
  • Contradiction detection between specialists
  • 3-tier priority triage (P1/P2/P3)
  • QA gate before final output

New delegation pattern

Old: Director → raw 31 specialists directly
New: Director → Strategist/Manager layer → specialists

Director now delegates to:
  • /google-strategist (for Google planning)
  • /google-manager (for Google execution)
  • /meta-strategist (for Meta planning)
  • /meta-manager (for Meta execution)
  • /creative-strategist (for creative work)
  • /cro-strategist (for landing pages)
  • /competitive (for intel)
  • /qa (always, before delivery)

Project Manager
/pm
NEW

Mode A: Internal sequencing (single client)

  • Enforces the dependency graph (tracking verified before bidding, keywords before copy)
  • Flags when a downstream task is blocked by an upstream blocker
  • Produces a sequenced task list with owners and dependencies marked
  • Prevents the team from doing work out of order

Mode B: Cross-client scheduling

  • Tracks all 17 clients: what's overdue, what's upcoming, what's in-flight
  • Surfaces which clients need a weekly check, monthly report, or campaign review this week
  • Flags clients that have gone quiet (no review in X days)
  • Produces a weekly agency workload view

Inputs

  • All client-info.md files (for deadlines, budgets, status)
  • Report history (last monthly report date, last weekly check)
  • Current task request from Marketing Director
  • Known blockers (tracking broken, campaign paused, etc.)

Outputs

  • Sequenced task list with dependency map
  • Cross-client status board (who needs what this week)
  • Blocker report (what's stuck and why)
  • Overdue items flagged with urgency level
Trigger phrases

"what needs to happen this week", "what's the order of operations for [client]", "which clients need attention", "sequence this project", "what's overdue"
